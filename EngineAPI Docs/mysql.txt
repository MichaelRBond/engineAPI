
EngineAPI Connector Functions:

########
dbConnect(command,value[,state])

command -- 
	- database : Used to connect to the database provided in "value." By default the connection will use the username, password, and host settings from EngineVars.
	- username : Override the default values from $engineVars 
	- password : Override the default values from $engineVars 
	- port     : Override the default values from $engineVars 
	- server   : Override the default values from $engineVars 

value -- string passed in for corresponding command. 

state -- BOOLEAN, defaults to FALSE. Only used wen the command is "database." If set to TRUE the variable $this->openDB is set to the database. If set to FALSE the database object is returned instead. 

Basic Usage:

$returnValue = $engine->dbConnect("database","myDatabase",TRUE);
if ($returnValue === FALSE) {
	//Handle Error
}

//$engine->openDB is now set with the database object.

$database = $engine->dbConnect("database","myDatabase");
if ($database === FALSE) {
	//Handle Error
}

// $database is now set with the database object

########
dbTables($name,$state[,$value])

name -- The 'friendly' name of the table. This is the human-friendly name of the table. Example: table "evlUserCnfg" could have a name of "Evans User Config"

state -- used to define different table sets. Useful for systems that have production, development, and/or testing tables. Any string to describe the type of table. example: "prod" for production or "dev" for development.

value -- OPTIONAL. if value is defined, dbTables sets value as the database table name for the name provided. If value is omitted the value for name is returned, FALSE if it hasn't been set yet. 

########
dbTablesExport()

Returns a array of all the defined tables for a database. This function is provided for compatibility with EngineAPI 1.0
